SECTION 4: MOTIVATION / BACKGROUND THEORY

Explain why Object-Oriented Programming (OOP) was chosen for the ShopSense application instead of procedural programming. 

Write these points in your background theory section:
OOP provides the perfect blueprint for building billing and credit tracking applications because real-world entities map directly to software objects. In ShopSense, the store keeper deals with concrete concepts like "Customers" and "Transactions". Creating dedicated classes for these concepts makes the code logical, reusable, and easy to maintain. 

Next, define how each OOP concept benefits ShopSense:
- Encapsulation: Prevents unauthorized modification of sensitive credit limits and balances by keeping class variables private and exposing them only through controlled getter and setter methods.
- Inheritance: Reduces code repetition. By creating a parent class like Person, we can reuse fields like name and phone number inside the Customer subclass without writing them twice.
- Polymorphism: Allows us to treat different transaction types (purchases vs. payments) uniformly while execution behavior changes dynamically depending on the active object.
- Abstraction: Simplifies complex operations. We can define abstract layouts for accounts and transactions, hiding complex calculations from the user interface.
- Interfaces: Creates strict behavioral rules. By using an Alertable interface, we guarantee that any customer class we implement in the future will contain credit limit validation.
- Exception Handling: Ensures the program does not crash when mistakes happen. It intercepts credit breaches or missing database records and shows friendly warning dialogs instead of error logs.
- File Handling: Guarantees that data is persistent. Saving credit records to text files ensures shopkeepers do not lose customer information when they close the program.


SECTION 5: PROBLEM STATEMENT

Describe the real-world operational challenges faced by kiryana store owners in Pakistan.

Detail the following problems in your report:
Small shopkeepers in Pakistan extensively sell goods on credit (udhaar) to regular local families to build customer loyalty. To track this, they use traditional paper pocketbooks or registers. This manual system has massive drawbacks:
1. Paper registers easily get torn, lost, wet, or destroyed.
2. Manually calculating balances for dozens of customers leads to mathematical errors, causing financial losses.
3. There is no real-time check to prevent a customer from exceeding their credit limit, leading to uncontrolled bad debts.
4. Generating history reports or receipt invoices manually is slow and tedious.

Then, explain the solution:
ShopSense digitally solves these problems by providing an automated desktop ledger. It calculates balances instantly, stores records safely on a storage device, alerts the merchant if a customer tries to buy past their credit limit, and generates printable invoices in a click.
